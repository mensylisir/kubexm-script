# KubeXM 命令追踪与参数矩阵设计

日期：2026-03-11

## 1. 目标与范围
本设计用于指导对当前仓库 `bin/kubexm` 的所有命令与参数进行**静态全量追踪**，形成：
- 按命令分章的调用链报告（追踪至最深实现函数/脚本/系统调用）
- 参数场景矩阵（命令行、环境变量、conf/、defaults.sh 的合并与优先级）
- 可靠性评估与风险清单（不改代码，仅分析与建议）

覆盖范围：
- `bin/kubexm` 中所有命令、子命令、选项
- 相关 `lib/`、`scripts/`、`conf/`、`defaults.sh` 等配置与执行路径

## 2. 总体结构（交付物）

### 2.1 命令分章调用链报告
- 每个命令/子命令独立成章
- 逐层追踪：命令解析 → 分发 → phase 脚本 → lib 函数 → 实际执行点
- 每个链路节点标注 `file:line`

### 2.2 参数场景矩阵
- 统一参数来源优先级：**命令行参数 > 环境变量 > conf/ > defaults.sh**
- 对每个命令建立“参数×场景”表
- 场景需覆盖：
  - kubernetes_type：kubeadm / kubexm（二进制）
  - etcd_type：kubeadm / kubexm（二进制）/ exists（跳过安装，仅配置）
  - loadbalancer：启用/禁用
  - loadbalancer_mode：external / internal / kube-vip / exists
  - loadbalancer_type：
    - external：kubexm_kh（keepalived+haproxy）/ kubexm_kn（keepalived+nginx）
    - internal：
      - haproxy + kubeadm → 静态 Pod haproxy
      - haproxy + kubexm → 二进制 haproxy
      - nginx + kubeadm → 静态 Pod nginx
      - nginx + kubexm → 二进制 nginx

### 2.3 可靠性评估与风险清单
对每条调用链逐项核查：
- 输入/配置校验
- 幂等性与可重复执行
- 失败处理与回滚
- 可观测性（日志/错误输出）
- 生产环境安全（权限/依赖/关键路径风险）
- 依赖可用性与版本漂移风险
- 版本兼容性耦合（K8s/kubeadm/etcd 等）
- 错误传播路径（`set -e`/`trap`/退出码）
- 并发/锁/竞态风险

## 3. 追踪方法

### 3.1 调用链追踪策略
- 入口：`bin/kubexm`
- 追踪类型：`source` 加载、`case` 分发、phase 脚本调用、lib 函数调用、系统执行点
- **source 侧效应**：记录 `source` 时触发的变量赋值/条件执行/默认值注入等副作用，并标注 `file:line`
- **conf/ 选择与加载顺序**：以代码中**显式顺序为准**；若使用 `glob`/目录遍历且未声明顺序，默认按**字典序**记录，并标注为“实现未明确顺序”。同级冲突按优先级覆盖规则处理。
- **动态执行点**：对 `${cmd}`/`eval`/`xargs` 等间接执行，记录为“动态终点”，列出候选命令与触发条件，并在风险清单中标注不确定性。
- 终点：最深的执行函数/脚本/系统命令
- 输出：链路节点标注 `file:line`

### 3.2 参数来源合并
- 合并优先级：命令行参数 > 环境变量 > conf/ > defaults.sh
- 需要明确覆盖/冲突规则和默认值

### 3.3 矩阵构建
- 按命令建立“参数×场景”表
- 场景与调用链路径建立映射
- 所有矩阵项可回溯到具体 `file:line`

## 4. 约束与原则
- 仅做静态分析与设计，不修改任何代码
- 报告必须可复核、可定位（`file:line`）
- 覆盖所有命令和参数（`bin/kubexm` 全量）

## 5. 备选方案与取舍（Brainstorming Summary）
- **方案 A：全量静态追踪（选定）**
  - 优点：覆盖全面、无需运行环境、可复核
  - 缺点：对动态分支/间接 `source` 的判断依赖代码可读性
- **方案 B：命令优先级分层追踪**
  - 优点：更快产出高价值内容
  - 缺点：无法满足“全量覆盖”的要求
- **方案 C：静态 + 轻量运行验证**
  - 优点：可验证动态路径
  - 缺点：需要执行环境，可能产生副作用

## 6. 假设与依赖
- 仓库结构稳定且 `bin/kubexm` 为统一入口
- 脚本中不大量依赖 `eval`/运行时生成的代码路径
- 允许只读读取 `conf/` 与 `defaults.sh` 等配置文件

## 7. 静态分析限制与边界情况
- 间接 `source`、变量拼接路径可能导致静态追踪歧义
- 环境变量动态覆盖可能导致分支路径不唯一
- 对此类情况在报告中明确标注为**不确定路径**并列出依赖条件

## 8. 参数矩阵完整性与冲突规则
- **完整性定义**：每个命令必须覆盖其**所有可见参数**（命令行、环境变量、conf/、defaults.sh），并为每个参数列出**取值范围/默认值/约束**与对应场景分支。对于“禁用+设置了 mode”等不一致配置，标注为冲突场景并进入风险清单。
- **参数识别范围**：
  - `bin/kubexm` 的 CLI 选项/子命令参数
  - `lib/` 与 `scripts/` 中显式读取的环境变量与配置键
  - `conf/` 配置文件键
  - `defaults.sh` 中的默认值变量
- **可选/派生参数**：标注为“派生”并指明来源（如由配置组合或脚本计算生成）。
- **冲突规则**：遵循优先级（命令行 > 环境变量 > conf/ > defaults.sh）。**仅显式提供的 CLI 标志覆盖下层**；未提供的 CLI 视为未设置。**空值统一视为未设置**（即不覆盖已有值），该规则优先于严格优先级，用于消除空值歧义。空值定义为：未设置、空字符串、仅空白字符；字符串 “null” 视为普通值。若同级多源冲突，记录为风险并列出条件。
- **CLI 重复与可选值**：重复标志以**最后一次出现**为准；`--flag=` 视为空值（即未设置）。如实现中出现空值作为显式覆盖的迹象，需标注为“待确认”并在风险清单中提示。短/长标志视为同义并在矩阵中合并展示。
- **位置参数**：视为 CLI 来源，需纳入矩阵并标注其含义与约束。
- **内部调用位置参数**：对 `bin/kubexm` 调用 phase 脚本/函数时传递的**位置参数**必须记录为参数项并标注 `file:line`。
- **环境变量分类**：由 `source` 产生的变量若仅作为默认值使用，归类为 defaults；**若变量在同一文件未赋值却被读取**，视为 ENV 输入。
- **空值语义核验**：若发现空值是否具备实际语义不确定，矩阵需同时记录“空值=未设置/空值=显式值”的双路径并标记风险。
## 9. 动态路径标记规范
- 变量拼接或条件 `source` 的不确定路径以 `?` 标识，并列出候选目标与触发条件。
- 报告中对每个 `?` 节点增加“条件说明”。

## 10. 输出格式（强制模板）
### 10.1 命令分章
每章必须包含：
- 命令/子命令
- 入口位置（`file:line`）
- 调用链（逐节点 `file:line`）
- 最深执行点（外部可执行或系统调用）
- 关键参数与来源摘要
- 风险与可靠性要点

### 10.2 命令/选项清单（覆盖核对）
- 需生成 `bin/kubexm` 的**完整命令/子命令/选项清单**，作为“全量一致性检查”的核对表。
- 清单条目需包含对应 `file:line` 或链接到命令分章中的 `file:line` 记录。

### 10.3 参数矩阵行
每行必须包含：
- 参数名
- 来源（CLI/ENV/conf/defaults/派生）
- 优先级
- 默认值
- 约束/互斥
- 关联场景
- 关联调用链节点（`file:line`）

## 11. 最深执行点边界规则
- 终止于**首次外部可执行**（非当前脚本内函数）。
- “系统调用”在本项目语境下等同于**外部命令执行**，包含对系统状态有影响的命令与重定向。
- Shell 内建（如 `echo`/`test`/`printf`）不作为终点，除非其行为构成关键状态变更（此时标注为“内建终点”。）
- 常见终点示例：`kubeadm`、`kubectl`、`etcd`、`systemctl`、`containerd`、`docker`、`cp`、`mv`、`sed`、`tar`、`curl` 等。

## 12. 验证方式
- **全量一致性检查**：确保所有命令/子命令/选项均在报告与矩阵中出现并可回溯到 `file:line`。
- **抽样复核（质量抽检）**：对关键命令样本进行链路抽样复核（每类命令至少 1 个样本：download/create/delete/scale/upgrade/renew/registry/health/rollback 等）。
- 关键配置项与矩阵条目必须能回溯到 `file:line`。

## 13. 验收标准
- 覆盖 `bin/kubexm` 全部命令/子命令/选项（含帮助/隐藏/废弃项，除非明确标注为内部保留且不可达）
- 每条调用链可追溯到最深执行点并标注 `file:line`
- 参数矩阵覆盖命令行、环境变量、conf/、defaults.sh 且给出优先级与冲突规则
- 风险清单覆盖输入校验、幂等性、回滚、可观测性、生产安全
